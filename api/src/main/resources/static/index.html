Buzzworthy Resource Locator (BRL) — REST API Operations:

User documentation available @ https://localhost:8443/BRL/api/

* Using old HTTP endpoints (requests to http://localhost:8080) will automatically be redirected to new HTTPS endpoints.

GET https://localhost:8443/BRL/api/wiki/verification

Purpose — Verify user credentials and retrieve authenticated user information.

Request header — "Bearer: abcdefg...tuvwxyz" where this string is an SSO token.

Request body — Empty.

Return body — {"firstName": "First", "lastName": "Last"}

Return code — 200 (successfully verified), 400 (misformatted request), 401 (unable to authenticate user), 405 (method not allowed for current endpoint), 500 (internal server error)

GET https://localhost:8443/BRL/api/wiki/about-us

Purpose — Retrieve the "About Us" page information.

Request header — None.

Request body — Empty.

Return body — JSON string containing the "About Us" page content.

Return code — 200 (successfully retrieved), 405 (method not allowed for current endpoint), 500 (internal server error)

Record Formats (JSON):

When sending to API:

{resource} = {
"title": "A meaningful title...",
"description": "A meaningful description...",
"url": "http://...",
}

{comment} = {
"contents": "A meaningful comment..."
}

{reviewFlag} = {
"contents": "A meaningful comment..."
}

{keywordList} = {"keywords": "keyword1, keyword2, keyword3, ..."}

When receiving from API:

Note: The creationDate field is returned as a timestamp (milliseconds since Unix epoch, January 1, 1970, 00:00:00 GMT). For example, 1733396445123 represents December 5, 2025, 10:07:25.123 GMT.
